Input Format

The input format expected by this class is an object of class de.pxlab.stat.FactorialDataTable. Such a table contains a single line for every data element. The first column of the table always is the random factor and the last column always is the data element. All other columns correspond to experimental factors and the respective entries in the table are factor level combinations. Here is an example data file from Keppel & Wickens (2004, Table 13.1):

The first colum is an identifier for the subject, the second and third column describe the levels of the experimental factors and the fourth column is the data element. Note that all but the last column may be non-numeric strings.

Output Format

The output is a HTML file. It contains three section:

  1. Cell Statistics. These are statistics of all cells of the design. There is a subsection for every source of variability in the design. As an example a two way analysis of variance has the following sources of variablity:
    • T the over all statistics
    • A the first factor
    • B the second factor
    • AB the interaction.
    Factors are named after the respective experimental parameter if these names are available. The table lists the following cell statistics for every level of the respective factor or factor combination:
    • N the number of levels of the random factor for the respective factor level combination.
    • Sum the sum of the data values for the respective factor level combination.
    • SumSQ the sum of the squared data values for the respective factor level combination.
    • [ ] the bracket term for the respective factor level combination. Bracket terms are defined by Keppel & Wickens (2004, pp 29ff).
    • Mean the mean of the data values for the respective factor level combination.
    • SD the standard deviation within the respective factor level combination.
    • SE the standard error of the mean for the respective factor level combination.
    Furthermore the respective subtable for each source also contains the bracket term for the respective source. The bracket term at a given factor level combination is the value of Sum*Sum/N for this factor level combination. The bracket term for the source is the sum of the bracket terms over all factor level combinations of the respective source. The bracket terms are used to calculate the sums of squares for various tests.
  2. Design Information. Describes the design of the data. This includes the number of factor levels for each factor and the type of each factor. The first column of the input data file always is the random factor and the last column always are data values. Between factors are identified by the fact that there exist levels of the random factor which are not crossed with levels of the between factor.
  3. Test Results. Gives the computational results for the statistical tests. It contains the bracket values, the sum of squares, the degrees of freedom for every sum, the mean squares, F-value and probability. It also contains a descriptive partial effect size statistic R2p and an estimate of the partial omega squared as defined by Keppel & Wickens (2004, p. 233f). The latter is only computed for tests involving only between factors.
